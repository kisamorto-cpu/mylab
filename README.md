# Лабораторная работа: Диаграммы классов/объектов с использованием языка UML

## Вариант: Продукты (фрукты, молочная продукция, мясо)

## Описание задачи
Разработать программу с иерархией наследования классов для продуктов питания.
В каждом классе должно быть не менее одного числового и одного строкового поля.
При вводе числовых параметров обязательна проверка на число и на диапазон.

## Иерархия классов

```
Product (базовый класс)
├── Fruit (Фрукты)
├── DairyProduct (Молочная продукция)
└── Meat (Мясо)
```

## Описание классов

### Product (Базовый класс)
**Поля:**
- `name: String` - название продукта (строковое)
- `price: double` - цена продукта (числовое, диапазон: 0-1000000)

**Методы:**
- Конструктор по умолчанию
- Конструктор с параметрами
- Геттеры и сеттеры с проверкой диапазона
- `equals()` - сравнение объектов
- `hashCode()` - хэш-код объекта
- `toString()` - строковое представление

### Fruit (Фрукты)
**Поля:**
- `origin: String` - страна происхождения (строковое)
- `weight: double` - вес в кг (числовое, диапазон: 0-100)

**Наследует:** все поля и методы от Product

### DairyProduct (Молочная продукция)
**Поля:**
- `fatContent: int` - процент жирности (числовое, диапазон: 0-100)
- `expirationDate: String` - срок годности (строковое)

**Наследует:** все поля и методы от Product

### Meat (Мясо)
**Поля:**
- `type: String` - вид мяса (строковое)
- `weight: double` - вес в кг (числовое, диапазон: 0-100)

**Наследует:** все поля и методы от Product

## Функциональность приложения

Консольное приложение с меню:

1. **Добавить новый элемент** - добавление продукта в коллекцию
   - Выбор типа продукта (Фрукт/Молочная продукция/Мясо)
   - Ввод названия и цены
   - Ввод специфических параметров для каждого типа
   - Проверка всех числовых значений на корректность и диапазон

2. **Удалить элемент по индексу** - удаление продукта из коллекции

3. **Вывод всех элементов** - отображение всех продуктов в терминале

4. **Сравнение двух элементов на равенство** - сравнение продуктов по индексам

5. **Завершение работы приложения**

## Структура проекта

```
src/main/java/com/products/
├── Product.java          # Базовый класс
├── Fruit.java            # Класс Фрукт
├── DairyProduct.java     # Класс Молочная продукция
├── Meat.java             # Класс Мясо
└── ProductApp.java       # Основное приложение

uml-diagram.puml          # Исходный файл UML диаграммы (PlantUML)
uml-diagram.png           # Сгенерированное изображение диаграммы
```

## Компиляция и запуск

```bash
# Компиляция
javac -d out src/main/java/com/products/*.java

# Запуск
java -cp out com.products.ProductApp
```

## UML Диаграмма

Диаграмма классов представлена в файлах:
- `uml-diagram.puml` - исходный код диаграммы на языке PlantUML
- `uml-diagram.png` - графическое представление диаграммы

Диаграмма показывает:
- Иерархию наследования (Product → Fruit, DairyProduct, Meat)
- Все поля классов с указанием типов и видимости
- Все методы классов с указанием параметров и возвращаемых типов
- Связи между классами
