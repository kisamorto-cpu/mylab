ОПИСАНИЕ ПРОЕКТА
----------------
Данная программа реализует консольное приложение для управления коллекцией 
продуктов с иерархией классов на основе базового класса Product.

ИЕРАРХИЯ КЛАССОВ
----------------
1. Product (базовый абстрактный класс)
   - Поля: name (строка), price (число)
   
2. Fruit (наследник Product)
   - Дополнительные поля: origin (строка), weight (число)
   
3. DairyProduct (наследник Product)
   - Дополнительные поля: fat_content (число), expiration_days (число)
   
4. Meat (наследник Product)
   - Дополнительные поля: animal_type (строка), protein (число)

5. ProductManager (класс для управления коллекцией)
   - Содержит коллекцию объектов типа Product

ФУНКЦИОНАЛЬНОСТЬ
----------------
Каждый класс реализует:
- Конструктор по умолчанию
- Конструктор с параметрами
- Метод equals() - сравнение объектов на равенство
- Метод hash_code() - вычисление хэш-кода
- Метод to_string() - строковое представление объекта
- Метод get_type() - получение типа продукта
- Статические методы валидации для числовых и строковых полей

МЕНЮ ПРИЛОЖЕНИЯ
---------------
1. Добавить новый элемент
   - Выбор типа продукта (Фрукт, Молочная продукция, Мясо)
   - Ввод всех необходимых параметров с проверкой на тип и диапазон
   
2. Удалить элемент по индексу
   - Ввод индекса элемента для удаления
   - Проверка корректности индекса
   
3. Вывод всех элементов
   - Отображение всей коллекции продуктов с индексами
   
4. Сравнение двух элементов на равенство
   - Ввод двух индексов
   - Сравнение соответствующих объектов через метод equals()
   
5. Завершение работы

ВАЛИДАЦИЯ ДАННЫХ
----------------
Все числовые параметры проходят проверку:
- На соответствие типу (число)
- На допустимый диапазон значений

Все строковые параметры проходят проверку:
- На соответствие типу (строка)
- На непустоту значения
